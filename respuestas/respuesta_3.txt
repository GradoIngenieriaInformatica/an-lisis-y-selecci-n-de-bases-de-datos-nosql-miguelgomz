TIPO | Justificación corta

En el caso de un sistema de métricas de servidores que maneja millones de registros indexados por fecha y hora, la mejor alternativa es una base de datos orientada a series temporales, como InfluxDB o TimescaleDB. Este tipo de bases de datos está diseñado específicamente para almacenar grandes volúmenes de datos que se generan de manera continua a lo largo del tiempo. Permiten realizar consultas eficientes por rangos de fechas, así como cálculos agregados como promedios, máximos, mínimos o análisis de tendencias. Además, están optimizadas para soportar altas tasas de escritura y compresión de datos históricos. Aunque una base de datos relacional podría utilizarse, no estaría optimizada para manejar este tipo de carga con la misma eficiencia que una base de datos especializada en series temporales.
